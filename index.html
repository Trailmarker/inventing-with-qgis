### anatomy of a QGIS plugin * a Python program that runs inside QGIS and can't really run without QGIS * accesses QGIS [link] and Qt [link] APIs to access and process data, make maps, add and remove layers, add and remove tools and toolbars, change symbology and do, well, almost anything * Python bootstrapped by QGIS when the plugin is activated

### a bit like app development? as with developing an app or an embedded system, the basic model is that you're building your code *outside of the thing it runs on*, then *transferring it to the thing to run* *this is slow and it's better if it's faster*
### making feedback faster (version 1) * experiment in the QGIS Python REPL * develop code that can be run outside of QGIS * write tests for this code * Plugin Reloader (also a plugin)
### making feedback faster (version 2) * integrate all testing with a full running version of QGIS * auto-reload the plugin when code changes * use QGIS Processing tools in your code
demo - auto-running unit tests in a QGIS plugin
### QGIS processing framework * provides a click-and-connect process modeller * access a ~~large~~ *vast* suite of standard QGIS processing tools * user interface to activate individual tools, with help * package tools with your plugin and use them on an ad hoc basis
### QGIS processing framework * QGIS processing tools can also access any FOSS4G library packaged in Python with QGIS (this is a lot of them) * this includes eg GDAL, OGR, OWSLib * this is *very powerful*
demo - exploratory testing with QGIS processing tools
### Bonus: Qt versus reactive front-end frameworks * Qt was invented to do event-driven cross-platform UX programming in C++ * many extremely cool innovations (signals, slots, layouts) * still a leading framework for cross-platform UX (competitors are things like Java, Electron, Tcl/Tk)
### a reactive web front-end … embedded in QGIS * it is actually possible to embed web frameworks in QGIS via the QWebView * the QWebView can host a full React, Angular, Aurelia, etc application written in JavaScript or TypeScript
